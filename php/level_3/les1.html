Stap 2 — Theorie

Hoe werkt PHP?

PHP draait op de server. De gebruiker ziet alleen het resultaat, nooit jouw code.

1
Jij typt een URL
localhost/php_lessen/index.php
localhost/php_lessen/index.php
2
Server leest het bestand
En voert de PHP code uit
<?php echo "Welkom!"; ?>
3
Resultaat terug
Alleen HTML, geen PHP code
Welkom!
👁

Rechts-klikken → "Paginabron weergeven" toont alleen HTML. Je PHP code is onzichtbaar voor bezoekers. Dit is het grote verschil met JavaScript!

Stap 3 — Variabelen

Variabelen

Een variabele is een opslag met een naam. In PHP begint elke variabele met $

variabelen.php
<?php

$naam = "GameMaster";   // tekst
$xp = 250;             // getal
$online = true;         // waar/onwaar

echo $naam;             // GameMaster
echo $xp;               // 250

?>
$naam
"GameMaster"
string
$xp
250
int
$online
true
bool
Stap 3 — Variabelen

Regels voor variabelen

Mag wel

$naam = "Tom";
$user_name = "Bo";
$xp2 = 100;
$_level = 5;

Begin met $ + letter of underscore.
Mag cijfers bevatten (niet als eerste).

Mag niet

$2naam = "Tom";  // start met cijfer
$mijn naam = "Bo"; // spatie
$mijn-xp = 100;  // streepje
naam = "test";   // geen $

Geen spaties, geen streepjes, niet beginnen met een cijfer, en altijd een $

⚠

Hoofdlettergevoelig! $Naam en $naam zijn twee verschillende variabelen. Wees consistent!

Even checken

Theorie check

Wat is de output van deze code?
$score = 100;
echo "Score: " . $score;
A: Score: $score
B: Score: 100
C: Error — je mist een puntkomma
D: Score:
De punt (.) plakt de tekst "Score: " en de waarde van $score (100) aan elkaar. Dubbele aanhalingstekens zouden hier ook werken: "Score: $score"
Waar draait PHP code?
A: In de browser, net als JavaScript
B: Op de server (bijv. XAMPP)
C: Op allebei
PHP draait alleen op de server. De server voert de code uit en stuurt het resultaat (HTML) naar de browser. De browser ziet nooit PHP code.
Even checken

Theorie check

Wat is de output van deze code?
$naam = "Tom";
echo 'Hallo $naam';
A: Hallo $naam
B: Hallo Tom
C: Error
Met enkele aanhalingstekens ('...') worden variabelen NIET vervangen. Je ziet letterlijk $naam. Met dubbele ("...") zou het wél "Hallo Tom" tonen. Dit is een veelgemaakte fout!
Wat gebeurt er als je een PHP-bestand opslaat als .html in plaats van .php?
A: Het werkt gewoon
B: De PHP code wordt niet uitgevoerd
C: Je krijgt een foutmelding
De server kijkt naar de bestandsextensie. Bij .html stuurt hij het bestand direct door. Bij .php voert hij eerst de code uit. Met .html zie je letterlijk <?php echo "test"; ?> in je browser.

Stap 5 — Zelfstandig werken

Individuele opdracht

Maak een nieuw bestand game.php in je php_lessen map.

Opdracht: Game Review pagina

1. Maak variabelen aan voor een game review:
    $game_naam, $genre, $score (1-10), $review, $aanbevolen (true/false)

2. Bouw een HTML pagina met een <h1> voor de game naam
    Toon het genre, de score en je review in <p> tags

3. Voeg minstens 2 extra variabelen toe naar eigen keuze
    (ontwikkelaar, release jaar, speeltijd, platform...)

4. Bonus: Gebruik strings samenvoegen om een zin te maken:
    echo "Ik geef " . $game_naam . " een " . $score . "/10";
🛠

Open http://localhost/php_lessen/game.php om je resultaat te zien. Probeer de score te veranderen en ververs de pagina!
