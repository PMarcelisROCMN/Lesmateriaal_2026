Stap 2 — Theorie

Hoe werkt PHP?

PHP draait op de server. De gebruiker ziet alleen het resultaat — nooit jouw code.

1
Jij typt een URL
localhost/gastenboek/index.php
localhost/gastenboek/index.php
2
Server leest het bestand
En voert de PHP code uit
<?php echo "Welkom!"; ?>
3
HTML terug naar browser
Alleen het resultaat, geen PHP
Welkom!
💡

PHP vs JavaScript: JavaScript draait in de browser. PHP draait op de server. Beide zijn mogelijk, maar voor databases heb je server-side nodig.

👁️

Rechts-klikken → "Paginabron weergeven" toont alleen HTML. Je PHP code is onzichtbaar voor bezoekers!

Even nadenken

Wat denk je dat deze code doet?

Bespreek met je buurman/buurvrouw. Nog niet uitproberen!

welkom.php
<!DOCTYPE html>
<html>
<body>
  <h1>Welkom</h1>
  <p>Datum: <?php echo date('d-m-Y'); ?></p>
</body>
</html>
Wat zie je in de browser als je dit bestand opent?
A: De PHP code zelf (met <?php erin)
B: "Welkom" als titel en de datum van vandaag
C: Alleen "Welkom" — de datum wordt niet getoond
D: Een foutmelding
De server voert date('d-m-Y') uit en vervangt dat stukje door de datum. De HTML-structuur blijft staan. In de paginabron zie je later nooit <?php — alleen het resultaat.

De HTML blijft gewoon staan. Alleen de stukken tussen <?php ?> worden uitgevoerd en vervangen door het resultaat. De rest van het bestand gaat ongewijzigd door.

Variabelen

Een variabele is een opslag met een naam. In PHP begint elke variabele met $

Stap 4 — Variabelen

Regels voor variabelen

✅ Mag wel

$naam = "Tom";
$gebruikers_naam = "Bo";
$xp2 = 100;
$_admin = true;

Begin met $ + letter of underscore.
Mag daarna ook cijfers bevatten.

❌ Mag niet

$2naam = "Tom"; // start met cijfer
$mijn naam = "Bo"; // spatie
$mijn-xp = 100; // streepje
naam = "test"; // geen $
⚠️

Hoofdlettergevoelig! $Naam en $naam zijn twee verschillende variabelen. PHP maakt dit onderscheid altijd. Wees consistent!

Stap 4 — Strings

Strings samenvoegen

In PHP plak je teksten aan elkaar met een punt (.)

strings.php
<?php
$voornaam = "Anna";
$achternaam = "Jansen";

// Manier 1: punt operator
echo $voornaam . " " . $achternaam;   // Anna Jansen

// Manier 2: variabelen in dubbele quotes
echo "Hallo $voornaam!";                   // Hallo Anna!

// Let op: enkele quotes werken NIET!
echo 'Hallo $voornaam!';                  // Hallo $voornaam!
?>
⚠️

Enkle vs. dubbele quotes: In "..." worden variabelen vervangen. In '...' zie je letterlijk de tekst — inclusief het dollarteken. Dit is de meest gemaakte fout bij beginners!

Even checken

Theorie check

Wat is de output van deze code?
$score = 100;
echo "Score: " . $score;
A: Score: $score
B: Score: 100
C: Foutmelding — je mist een puntkomma
D: Score:
De punt (.) plakt "Score: " en de waarde van $score (100) aan elkaar. Het resultaat is "Score: 100". Alternatief: "Score: $score" met dubbele quotes werkt ook.
Wat is de output van: echo 'Hallo $naam'; als $naam = "Tom"?
A: Hallo Tom
B: Hallo $naam
C: Foutmelding
Met enkele aanhalingstekens worden variabelen NIET vervangen. Je ziet letterlijk $naam. Gebruik dubbele aanhalingstekens als je een variabele in een string wilt invoegen.
Even checken

Theorie check

Waar draait PHP code?
A: In de browser, net als JavaScript
B: Op de server (bijv. XAMPP)
C: Op allebei tegelijk
D: In het besturingssysteem van de gebruiker
PHP draait alleen op de server. XAMPP is je lokale server. De browser stuurt een verzoek, de server voert de PHP code uit en stuurt alleen HTML terug.
Wat gebeurt er als je een PHP-bestand opslaat als .html in plaats van .php?
A: Het werkt gewoon
B: De PHP code wordt niet uitgevoerd — je ziet de code letterlijk
C: Je krijgt een foutmelding van de server
De server kijkt naar de bestandsextensie. Bij .html stuurt hij het bestand direct door zonder het uit te voeren. Bij .php voert hij eerst de PHP code uit. Met .html zie je <?php echo "test"; ?> letterlijk in je browser.
